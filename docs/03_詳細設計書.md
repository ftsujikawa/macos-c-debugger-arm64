# 詳細設計書

## 1. モジュール詳細

---

### 1.1 main.c

#### 処理フロー

```
main(argc, argv)
  1. argc < 2 → usage() 表示してエラー終了
  2. cdbg_resolve_program(argv[1]) : カレントディレクトリ優先でパス解決
  3. spawn_argv[] に argv[1..] を格納（最大 256 要素）
  4. cdbg_init(&dbg)
  5. cdbg_set_run_target(&dbg, spawn_argv)
  6. cdbg_run(&dbg, NULL) : プログラムを起動してエントリで停止
  7. cdbg_repl(&dbg) : REPL ループ開始
  8. EXIT_SUCCESS で終了
```

---

### 1.2 process.c

#### cdbg_resolve_program

- 名前に `/` が含まれる → パスとして扱いそのままコピー
- 含まれない → `./<name>` が存在すればそちらを使用
- それ以外 → `PATH` 検索はせずそのまま（`execv` に委ねる）

#### cdbg_process_spawn

```
fork()
  子プロセス:
    1. setpgid(0, 0)       ← debuggee を独立プロセスグループへ
    2. ptrace(PT_TRACE_ME) ← 自身をトレース対象に登録
    3. malloc_stack_logging が true なら setenv("MallocStackLogging", "1")
    4. execv(argv[0], argv)

  親プロセス:
    *child_pid = pid; return 0
```

#### cdbg_process_wait

```c
do {
    ret = waitpid(pid, status, 0);
} while (ret == -1 && errno == EINTR);  // SIGINT による中断を再試行
```

### 1.3 memory.c

#### アクセス経路の優先順位

```
cdbg_mem_read(pid, addr, buf, len)
  1. task_for_pid() で Mach タスクポート取得
  2. 成功 → mach_vm_read() でまとめて読み取り
  3. 失敗 → ptrace(PT_READ_D) で 1 バイトずつ読み取り

cdbg_mem_write(pid, addr, buf, len)
  1. task_for_pid() で Mach タスクポート取得
  2. 成功 → mem_write_mach()
       2a. mach_vm_write() 直接試行
       2b. 失敗（保護あり）→ mach_vm_protect(R|W|COPY) → write → protect(R|X)
  3. 失敗 → ptrace(PT_WRITE_D) で 1 バイトずつ書き込み
```

**理由**: `mach_vm_write` は書き込み保護ページに対して失敗するため、`mach_vm_protect` で一時的に書き込み可能にする。これはブレークポイント（テキストセクション書き換え）に必要。

### 1.4 regs.c

#### レジスタ取得

```
cdbg_regs_get(pid, regs)
  1. task_for_pid() → task
  2. task_threads(task) → threads[0]（プライマリスレッドのみ対象）
  3. thread_get_state(thread, CDBG_THREAD_FLAVOR, &regs->native, &count)
  4. thread_get_state(thread, CDBG_NEON/FP_FLAVOR, &regs->neon/fp, &count)
     （同じ thread ポートを使い、deallocate は両呼び出しの後に行う）
```

#### アーキテクチャ抽象化

| 操作 | ARM64 | x86_64 |
|---|---|---|
| PC 取得 | `regs->native.__pc` | `regs->native.__rip` |
| FP 取得 | `regs->native.__fp` | `regs->native.__rbp` |
| SP 取得 | `regs->native.__sp` | `regs->native.__rsp` |
| トラップバイト | `0xd4` (BRK) | `0xcc` (INT3) |

#### レジスタ一覧

**ARM64**

| 種別 | レジスタ | Mach 構造体フィールド |
|---|---|---|
| 汎用 (GPR) | x0-x28, fp(x29), lr(x30), sp, pc, cpsr | `arm_thread_state64_t.__x[]`, `.__fp`, `.__lr`, etc. |
| NEON/FP | v0-v31 | `arm_neon_state64_t.__v[]` (`__uint128_t`) |
| FP 制御 | fpsr, fpcr | `arm_neon_state64_t.__fpsr`, `.__fpcr` |

**x86_64**

| 種別 | レジスタ | Mach 構造体フィールド |
|---|---|---|
| 汎用 (GPR) | rax-rsp, r8-r15, rip, rflags, cs, fs, gs | `x86_thread_state64_t.__rax` 等 |
| SSE | xmm0-xmm15 | `x86_float_state64_t.__fpu_xmm0`-`.__fpu_xmm15` (`_STRUCT_XMM_REG`) |
| SSE 制御 | mxcsr | `x86_float_state64_t.__fpu_mxcsr` |
| x87 FPU | st0-st7 | `x86_float_state64_t.__fpu_stmm0`-`.__fpu_stmm7` (`_STRUCT_MMST_REG`) |
| x87 制御 | fctrl, fstat, ftag | `x86_float_state64_t.__fpu_fcw`, `.__fpu_fsw`, `.__fpu_ftw` |

#### 新規関数

| 関数 | 説明 |
|---|---|
| `cdbg_regs_get_by_name(regs, name, out)` | レジスタ名で値を取得（整数として返す） |
| `cdbg_regs_set_by_name(pid, regs, name, value, is_float, fvalue)` | レジスタ名で値を設定し thread_set_state で書き戻す |
| `double_to_80bit(d, bytes)` | IEEE 754 double → 80ビット拡張精度（x87 st レジスタ用） |

#### フレームアンワインド (cdbg_regs_frame_up)

```
x86_64:
  1. PC が PUSH RBP (0x55) ならスタックトップの戻りアドレスを使用
  2. FP != 0 → [FP+0]=saved_FP, [FP+8]=ret_addr を読んで移動

ARM64:
  1. LR（リンクレジスタ）が有効なら PC を LR に設定
  2. FP != 0 → [FP+0]=saved_FP, FP を更新
  3. SP を FP+16 に設定
```

### 1.5 breakpoint.c

#### cdbg_bp_enable

```
1. cdbg_mem_read(pid, addr, &saved_byte, 1)
2. bp->addr = addr; bp->saved_byte = saved_byte; bp->enabled = true
3. cdbg_mem_write(pid, addr, &TRAP_BYTE, 1)
```

#### cdbg_bp_matches_pc

```c
// x86_64: INT3 実行後に IP が +1 されるため
#if defined(__x86_64__)
return bp->addr + 1 == pc;
#else
return bp->addr == pc;
#endif
```

#### ブレークポイントヒット後の再開処理（debugger.c 内）

```
1. cdbg_bp_disable() : 元のバイトを復元
2. cdbg_regs_set_pc(pc - 1) : PC を bp->addr に戻す（x86_64 のみ）
3. cdbg_regs_set() : PC をデバッギに書き戻す
4. シングルステップで 1 命令実行
5. cdbg_bp_enable() : トラップ命令を再設置
```

### 1.6 syms.c

#### シンボルロード戦略

```
cdbg_syms_load(executable_path)
  1. Mach-O をメモリマップ → __TEXT vmaddr を link_base として保存
  2. [executable_path, dSYM パス] それぞれに対して:
       syms_load_from_nm() : nm -n <path> の出力をパース
       成功したら return 0
  3. 全失敗 → Mach-O nlist_64 を直接パース (syms_load_macho)
  4. dSYM が存在すればそちらも試みる
```

#### シンボルフィルタリング条件 (sym_name_interesting)

- 名前が空でない
- type が `?` または `-` でない
- 先頭文字が `.` または `L` でない（ローカルラベル除外）
- 先頭文字が印字可能文字

#### シンボル名検索 (cdbg_syms_lookup_name)

```
入力名が '_' で始まらない場合、先頭に '_' を付けて検索
（C の external シンボルは Mach-O で先頭 _ が付く慣習）
```

#### ASLR スライド算出 (cdbg_syms_update_slide)

```
mach_vm_region_recurse で VM マップを先頭から走査:
  実行可能(EXECUTE|READ) な最初のリージョン先頭 = 実行時 __TEXT 先頭
  slide = 実行時先頭 - link_base
```

### 1.7 lineno.c

#### DWARF 行情報ロード

`dwarfdump --debug-line <executable>` の出力をパースして `cdbg_line_entry_t[]` を構築する。

```
cdbg_line_entry_t {
    uintptr_t address;   // リンク時アドレス
    uint32_t  line;
    uint32_t  column;
    char      file[512];
}
```

行情報のスライドは `cdbg_lineno_update_slide` で `cdbg_syms_t` と同様に算出。

#### 主要ルックアップ

| 関数 | 用途 |
|---|---|
| `cdbg_lineno_lookup_pc` | runtime_pc → line_entry |
| `cdbg_lineno_lookup_line` | file + 行番号 → line_entry |
| `cdbg_lineno_lookup_line_unique` | 行番号のみ（ファイル1つの場合） |
| `cdbg_lineno_lookup_next_line_after_pc` | ステップオーバー終端検出 |

### 1.8 expr.c

#### 式パーサ

演算子優先順位は C の規則に従う再帰降下パーサ。

```
parse_expression
  └─ parse_logical_or (||)
       └─ parse_logical_and (&&)
            └─ parse_bit_or (|)
                 └─ parse_bit_xor (^)
                      └─ parse_bit_and (&)
                           └─ parse_equality (== !=)
                                └─ parse_relational (< > <= >=)
                                     └─ parse_shift (<< >>)
                                          └─ parse_additive (+ -)
                                               └─ parse_multiplicative (* / %)
                                                    └─ parse_unary (+ - ! ~ & *)
                                                         └─ parse_primary (数値/変数/括弧)
```

#### サポートするトークン

数値リテラル（10進・16進・浮動小数点数）、識別子（変数名）、演算子全般、`.` `->` `[n]`（メンバアクセス）

#### 浮動小数点数リテラル

`strtoull` は `.` で止まるため、整数パース後に `*end == '.' | 'e' | 'E'` を検出したら `strtod` に切り替える。
`.5` のように `.` から始まるリテラルも同様に処理する。

```c
cdbg_expr_result_t {
    uint64_t value;      // 整数値、またはfloatのビットパターン (memcpy)
    double   fvalue;     // is_float == true の場合の double 値
    bool     is_float;   // 浮動小数点リテラルとして解析された場合 true
    bool     is_address;
    char     type[128];
}
```

単項マイナス (`-`) は `is_float` を引き継ぎ、`fvalue` を符号反転して `value` を再計算する。

#### ローカル変数解決 (find_local_var)

```
dwarfdump --debug-info <dSYM> の出力を解析:

  1. DW_TAG_subprogram を発見
  2. DW_AT_low_pc / DW_AT_high_pc で PC が関数内かチェック
  3. DW_TAG_variable / DW_TAG_formal_parameter を発見
  4. DW_AT_name, DW_AT_type, DW_AT_location (DW_OP_fbreg <offset>) を収集
  5. 対象変数の名前が一致したら out に返す
```

変数アドレス = FP + fbreg_offset

#### 型サイズ推定

DWARF の型名文字列から推定（`char`=1, `short`=2, `int`/`bool`=4, `long`/ポインタ=8）

### 1.9 debugger.c

#### REPL ループ

```c
for (;;) {
    fputs("cdbg> ", stdout); fflush(stdout);
    if (fgets(line, sizeof(line), stdin) == NULL) {
        if (ferror(stdin) && errno == EINTR) {
            clearerr(stdin); putchar('\n'); continue;  // Ctrl+C
        }
        break;  // EOF → 終了
    }
    // コマンドディスパッチ
}
```

#### コマンドディスパッチ一覧

| 入力 | 処理関数 |
|---|---|
| `help` / `h` | `cmd_help` |
| `run` | `cmd_run` |
| `continue` / `c` | `cdbg_continue` + `cdbg_wait` |
| `step` / `s` | `cdbg_step_next_line` |
| `si` | `cdbg_single_step` + `cdbg_wait` |
| `next` / `n` | `cdbg_next_source_line` |
| `up` | `cdbg_frame_up` |
| `kill` | `stop_debuggee`（REPL は継続） |
| `regs` / `r` | `cdbg_refresh_regs` + `cdbg_regs_print` |
| `print` / `p` / `p/fmt` | `cmd_print_from_repl` |
| `set` | `cmd_set` |
| `show` | `cmd_show` |
| `tb` | `cmd_backtrace` |
| `leaks` | `cmd_leaks` |
| `break` / `b` | `cmd_break` |
| `del` / `delete` | `cmd_del` |
| `dis` | `cmd_dis` |
| `list` / `l` | `cmd_list` |
| `lines` | `cmd_lines` |
| `lists` | `cdbg_lineno_print_list` |
| `syms` / `sym` | `cdbg_syms_print_list` |
| `x` | `cmd_examine` |
| `quit` / `q` | `stop_debuggee` + `cdbg_lineno_free` + `cdbg_syms_free` |

#### ステップ制御

**step (ソース行ステップイン)**

```
cdbg_step_next_line:
  current_line = lineno_lookup(pc)
  ループ:
    cdbg_single_step → cdbg_wait
    new_line = lineno_lookup(new_pc)
    new_line != current_line → 停止
    ブレークポイントにヒット → 停止
```

**next (ソース行ステップオーバー)**

```
cdbg_next_source_line:
  target_line = lineno_lookup_next_line_after_pc(pc)
  target_addr に一時ブレークポイントを設置 → continue → 到達したら削除
  ブレークポイント未解決の場合は step と同様のシングルステップループ
```

#### leaks コマンド (cmd_leaks)

```
1. malloc_stack_logging が false → 警告を出力（スタックトレースなしで続行）
2. snprintf(cmd, "leaks %d 2>&1", pid)
3. popen(cmd, "r") で leaks 出力をそのまま stdout に転送
4. leaks が見つからない場合（"leaks not found" 出力）はエラーメッセージ
```

#### 停止時の PID/TID 表示

ブレークポイントヒット・ステップ完了など、プロセスが停止するたびに以下を表示する：

```
[PID: 12345  TID: 67890]
```

- `get_primary_thread_id(pid)` : Mach `THREAD_IDENTIFIER_INFO` フレーバーで `thread_id`（起動ごとにユニークな64bit値）を取得
- `print_stop_header(dbg)` : PID + TID を `printf("[PID: %d  TID: %llu]\n", ...)` で出力
- 呼び出し箇所: `report_stop`, `cdbg_step_next_line`, `cdbg_next_source_line`, 初回エントリ停止（`cdbg_run` 内）

その後、ソースコードまたは逆アセンブルを `print_stop_location` で表示する（ソース情報がある場合はソース、ない場合は `dis`）。

#### set コマンド (cmd_set)

```
"var = expr" 形式をパース:
  1. 左辺が '$' で始まる → レジスタ代入モード
       a. cdbg_expr_eval で右辺を評価（is_float, fvalue を含む）
       b. cdbg_refresh_regs でレジスタ読み込み
       c. cdbg_regs_set_by_name(pid, regs, reg_name, value, is_float, fvalue) で書き込み
  2. それ以外 → 変数代入モード
       a. 左辺の lvalue パスを解析（変数名、.member、->member、[n]）
       b. cdbg_resolve_lvalue_expr で書き込みアドレスを解決
       c. cdbg_expr_eval で右辺を評価
       d. 型サイズに応じて cdbg_mem_write
```

#### help コマンド (cmd_help)

```
cdbg_help_entry_t {
    const char *category;  // "General" / "Execution" / "Inspection" / "Breakpoints" / "Settings"
    const char *names;     // "break, b"
    const char *usage;     // "break <addr|name|file:line|line>"
    const char *summary;   // 1行の概要
    const char *detail;    // NULL または複数行の詳細テキスト
}
```

| 関数 | 役割 |
|---|---|
| `print_help_entry` | `help` 一覧用：`  %-22s  %s\n` で names + summary を1行表示 |
| `print_help_entry_detail` | `help <cmd>` 用：usage + summary + detail（2スペースインデント）を表示 |
| `print_help_all` | カテゴリ見出し + 全コマンド1行一覧 + 式サンプル集 |
| `help_name_matches` | `names` フィールドのカンマ区切りトークンまたは `usage` 先頭との前方一致で検索 |

全コマンドに `detail` テキストを持つ。`help <cmd>` で書式例・サブコマンド・注意事項を確認できる。

#### kill コマンド

```
1. pid <= 0 or state == IDLE → "No process is running." を stderr に出力して return
2. killed_pid = dbg->pid を保存
3. stop_debuggee(dbg) : ptrace(PT_KILL) → cdbg_process_wait → pid=0, state=IDLE
4. printf("Process %d killed.\n", killed_pid)
5. REPL を継続（debugger は終了しない）
```

`quit` との違い: `kill` はデバッギを終了するが REPL に戻る。`quit` はデバッギを終了したうえでデバッガ自体も終了する。

#### stop_debuggee（quit / kill 時に呼ばれる）

```
1. pid <= 0 or state == IDLE → return 0
2. ptrace(PT_KILL, pid) : SIGKILL 相当でプロセスを終了
3. cdbg_process_wait() : zombie 回収
4. pid = 0; state = IDLE; breakpoints すべて disabled
```

## 2. エラー処理方針

- すべての関数は `0` を成功、`-1` をエラーとして返す（`bool` 返しを除く）
- ptrace・Mach API のエラーは `perror` または `fprintf(stderr, ...)` で報告
- REPL コマンドのエラーは stderr に出力し、REPL を継続する
- `cdbg_continue` / `cdbg_wait` がエラーを返した場合は REPL を終了する（`return -1`）
