# 基本設計書

## 1. システム概要

`cdbg` は macOS 上で動作する対話型 C デバッガである。`ptrace` と Mach カーネル API を組み合わせて debuggee を制御し、DWARF デバッグ情報を解析することで変数参照・ソース表示・ブレークポイントを実現する。

## 3. モジュール一覧

| モジュール | ファイル | 責務 |
|---|---|---|
| メインエントリ | `main.c` | 引数解析、`cdbg_t` 初期化、REPL 起動 |
| デバッガコア | `debugger.c` | REPL、コマンドハンドラ本体、ステップ制御、SIGINT ハンドラ、DWARF変数/型解決 |
| プロセス管理 | `process.c` | fork/exec、`waitpid`、プログラムパス解決 |

| レジスタ操作 | `regs.c` | Mach スレッド状態取得・設定、PC/FP/SP 抽象化、フレームアンワインド、デバッグ/例外状態取得 |
| ブレークポイント | `breakpoint.c` | トラップ命令（BRK）埋め込み/復元 |
| ウォッチポイント | `watchpoint.c` | ARM64 ハードウェアデバッグレジスタ（DBGWVR/DBGWCR）の設定、ESR からの発火要因判定 |
| シンボル管理 | `syms.c` | Mach-O シンボルテーブル解析、`nm` フォールバック、ASLR スライド算出 |

| コマンド解析 | `command.y` / `command.l` | REPL 入力行の構文解析(bison)・字句解析(flex)。旧 `strtok` チェーンを置き換え |
| 式評価器 | `expr.y` / `expr.l` / `expr_eval.c` | 式の構文解析(bison)・字句解析(flex)・AST評価（右辺値/左辺値の2モード） |

## 4. 状態機械

debugger は debuggee の状態を `cdbg_state_t` で管理する。

```
      run/spawn
  ┌─────────────────────────────────────────┐
  │                                         ▼
IDLE ◀────────────── exited/killed ──── RUNNING
  │                                         │
  │                                    waitpid停止
  │                                         │
  └──────────────── kill ──────────── STOPPED
                                            │
                           continue/step ───┘
```

| 状態 | 値 | 意味 |
|---|---|---|
| `CDBG_STATE_IDLE` | 0 | 対象プロセスなし |
| `CDBG_STATE_RUNNING` | 1 | 対象プロセス実行中 |
| `CDBG_STATE_STOPPED` | 2 | ptrace 停止中（ブレークポイント・シグナル等） |

## 5. 主要データ構造

### 5.1 cdbg_t（デバッガ中心構造体）

```c
typedef struct cdbg {
    pid_t            pid;                         // debuggee の PID
    cdbg_state_t     state;                       // 実行状態
    int              wait_status;                 // waitpid ステータス

    cdbg_syms_t      syms;                        // シンボルテーブル
    char             executable_path[1024];
    char             debug_info_path[1024];       // dSYM パス
    char             run_argv_storage[64][1024];  // 実行引数ストレージ
    char            *run_argv[65];
    size_t           run_argc;
    bool             print_pretty;               // pretty print フラグ
    bool             malloc_stack_logging;        // MallocStackLogging フラグ
    cdbg_language_t  language;                   // 式評価言語
} cdbg_t;
```

## 6. 起動シーケンス

```
main()
  │
  ├─ cdbg_resolve_program()   : パス解決
  ├─ cdbg_init()              : 構造体初期化
  ├─ cdbg_set_run_target()    : argv 保存
  ├─ cdbg_run()               : デフォルト起動（main 直後で停止）
  │    ├─ cdbg_load_symbols() : シンボル・行情報ロード
  │    ├─ cdbg_spawn()        : fork/exec → waitpid
  │    └─ entry まで実行して停止
  └─ cdbg_repl()              : REPL ループ（SIGINT ハンドラ登録）
```

## 7. ブレークポイント機構

### 設定

1. 対象アドレスの命令（4 バイト）を `cdbg_mem_read` で退避
2. `BRK #0`（`0xd4200000`）を書き込む
3. `cdbg_breakpoint_t` に {addr, saved_inst, enabled=true} を記録

### ヒット時

1. `waitpid` が停止を通知
2. BRK 実行後は PC が命令の直後を指すため、ブレークポイントのアドレスまで戻す
3. トラップ命令を元の命令に復元
4. ブレークポイントを一時無効化 → シングルステップで元の命令を実行 → 再有効化

## 7b. ウォッチポイント機構

`watchpoint.c` が ARM64 のハードウェアデバッグレジスタ（`DBGWVRn_EL1` / `DBGWCRn_EL1`、最大 4 本）を用いて実装する。ソフトウェア的なメモリ比較やシングルステップは使わない。

### 設定 (`cdbg_watch_arm`)

1. `thread_get_state(ARM_DEBUG_STATE64)` で現在のデバッグレジスタ状態を取得
2. 対象アドレスを 8 バイト境界に整列した値を `WVR` に設定
3. `WCR` に enable・特権レベル(EL0のみ)・読取/書込選択(LSC)・バイトマスク(BAS)を設定
4. `thread_set_state` で書き戻し、読み戻して実装されているレジスタ本数を検証（未実装なら失敗を返す）

### 発火時

ARM64 のウォッチポイント例外は対象命令の実行前に発生し、PC は命令実行後まで進まないため、そのまま `continue` すると同じ命令で無限に再トラップする。そこで発火のたびに:

1. `thread_get_state(ARM_EXCEPTION_STATE64)` で `FAR`（フォールトアドレス）と `ESR`（例外症候群レジスタ）を取得
2. `ESR` の EC フィールドがウォッチポイント例外（`0x34`/`0x35`）であることを確認し、`WnR` ビットで読み/書きを判定
3. `FAR` を各ウォッチポイントのアドレス範囲と照合してヒットしたものを特定
   （複数ウォッチポイントが隣接し、幅の広いロード/ストア命令が一括でまたぐ場合は特定できないことがあるため、その場合は「特定不能」として扱いつつ処理は継続する）
4. 全ウォッチポイントを一旦無効化 → シングルステップで対象命令を実際に実行 → 再度有効化

書き込みウォッチポイントの Old/New 値は、この手順の前後でメモリを読むことで取得する（前＝Old、シングルステップ後＝New）。

## 8. メモリアクセス戦略

```
cdbg_mem_read / cdbg_mem_write
  │
  ├─ task_for_pid 成功 → mach_vm_read / mach_vm_write（高速）
  │    書き込み時: 保護 (R+W) を一時変更して書き込み後に (R+X) に戻す
  │
  └─ 失敗 → ptrace PT_READ_D / PT_WRITE_D（1 バイトずつ）
```

## 9. シンボル・ASLR 対応

```
cdbg_syms_load()
  ├─ Mach-O をメモリマップ → __TEXT セグメントのリンク時ベースアドレスを取得
  ├─ nm コマンドでシンボルテーブルを文字列解析（第一優先）
  └─ 失敗時は Mach-O nlist_64 を直接パース

cdbg_syms_update_slide()
  └─ mach_vm_region_recurse で実行可能領域先頭を探索 → slide = 実行時先頭 - link_base

runtime_addr = link_addr + slide
```

## 10. DWARF 行情報

```
cdbg_lineno_load()
  └─ dwarfdump --debug-line を実行して出力をパース
       → cdbg_line_entry_t[] {address, line, column, file}

cdbg_lineno_update_slide()
  └─ シンボルと同様に slide を設定

runtime_pc → (pc - slide) → lookup → file:line
```

## 11. シグナル処理

```
SIGINT 受信（Ctrl+C）
  │
  ├─ state == RUNNING → kill(pid, SIGINT) で debuggee に転送
  │     → ptrace がシグナルを捕捉 → waitpid 復帰 → STOPPED 状態
  │
  └─ state != RUNNING（プロンプト待機中）
        → fgets が EINTR で中断 → clearerr + 改行 + プロンプト再表示

子プロセス分離:
  cdbg_process_spawn で setpgid(0, 0) を呼び、
  debuggee を独立したプロセスグループに移す。
  端末の Ctrl+C が両者に届かないよう分離した上で、
  シグナルハンドラが明示的に転送する。
```
