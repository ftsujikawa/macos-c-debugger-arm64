# テスト仕様書

## 1. テスト概要

### 1.1 テスト対象

`cdbg` の全コマンドおよびコア機能。テストは `build/target`（`examples/target.c` のビルド成果物）を debuggee として使用する。

### 1.2 テスト環境

| 項目 | 内容 |
|---|---|
| OS | macOS 13 以降 |
| アーキテクチャ | ARM64 (Apple Silicon) |
| ビルド | `make examples` |
| 実行 | `build/cdbg build/target` |

### 1.3 テスト種別

| 種別 | 内容 |
|---|---|
| 正常系 | 正しい入力に対して期待通りの出力が得られること |
| 異常系 | 不正な入力や未対応操作に対してエラーが適切に報告されること |

## 3. 確認観点チェックリスト

### ビルド

- [ ] `make` で警告・エラーなしでビルドできる（bison/flex の生成コードを含む）
- [ ] ARM64 でビルド・実行できる

### プロセス制御

- [ ] `run` → `continue` でプログラムが完走して終了報告される
- [ ] `run` → `run` で再起動できる
- [ ] `step` で関数呼び出し内部に入れる
- [ ] `next` で関数呼び出しをスキップできる

### ブレークポイント

- [ ] 関数名・ファイル:行・行番号の 3 種類で設定できる
- [ ] ブレークポイント通過後に再度ヒットする（正しく再設置される）
- [ ] del で削除したブレークポイントはヒットしない

### メモリアクセス

- [ ] print でローカル変数値が正しく表示される
- [ ] set で書き換えた値が print で確認できる
- [ ] x コマンドで指定アドレスのバイト列が表示される

### シグナル

- [ ] 実行中に Ctrl+C でデバッガが死なず debuggee が止まる
- [ ] プロンプト中に Ctrl+C でデバッガが死なずプロンプトに戻る
- [ ] quit でデバッガ終了後に debuggee の zombie プロセスが残らない
