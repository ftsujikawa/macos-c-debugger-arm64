# 要件定義書

## 1. 目的

本書は macOS 向け対話型 C デバッガ `cdbg` の要件を定義する。

---

## 2. スコープ

| 項目 | 内容 |
|---|---|
| システム名 | cdbg (C debugger for macOS) |
| 対象 OS | macOS (Apple Silicon / Intel) |
| 対象言語 | C（主）、C++（補助的サポート） |
| 提供形態 | コマンドラインツール |

---

## 3. 用語定義

| 用語 | 定義 |
|---|---|
| debugger | cdbg 本体プロセス |
| debuggee | cdbg によってデバッグされる対象プログラム |
| ブレークポイント | 実行を一時停止させる命令埋め込み点 |
| ptrace | macOS カーネルが提供するプロセストレース API |
| DWARF | デバッグ情報フォーマット（ELF/Mach-O に埋め込まれる） |
| dSYM | macOS の外部デバッグシンボルバンドル |
| slide | ASLR によるメモリアドレスとリンク時アドレスの差分 |

---

## 4. 機能要件

### 4.1 プロセス制御

| ID | 要件 |
|---|---|
| F-PC-01 | debuggee を起動できる（引数付き） |
| F-PC-02 | debuggee を再起動できる |
| F-PC-03 | 実行中の debuggee を再開できる（continue） |
| F-PC-04 | ソース行単位でステップ実行できる（step in） |
| F-PC-05 | ソース行単位でステップ実行できる（step over） |
| F-PC-06 | マシン命令単位でステップ実行できる（single step） |
| F-PC-07 | debugger が SIGINT を受けたとき、実行中の debuggee に SIGINT を転送する |
| F-PC-08 | quit 時に debuggee を強制終了してから debugger を終了する |

### 4.2 ブレークポイント

| ID | 要件 |
|---|---|
| F-BP-01 | アドレス指定でブレークポイントを設定できる |
| F-BP-02 | シンボル名（関数名）指定でブレークポイントを設定できる |
| F-BP-03 | `ファイル名:行番号` 指定でブレークポイントを設定できる |
| F-BP-04 | 行番号のみ指定でブレークポイントを設定できる |
| F-BP-05 | ブレークポイントを番号指定で削除できる |
| F-BP-06 | ブレークポイントをすべて削除できる |
| F-BP-07 | ブレークポイント一覧を表示できる |
| F-BP-08 | 最大 64 個のブレークポイントを管理できる |

### 4.3 状態参照

| ID | 要件 |
|---|---|
| F-IS-01 | レジスタの値を表示できる |
| F-IS-02 | 変数の値を式として評価・表示できる（`print`/`p`） |
| F-IS-03 | 出力フォーマットを指定できる（`/d` `/x` `/o` `/t` `/c` `/s`） |
| F-IS-04 | ローカル変数・引数・グローバル変数を一覧表示できる |
| F-IS-05 | 任意アドレスのメモリを 16 進ダンプで表示できる（`x`） |
| F-IS-06 | バックトレース（コールスタック）を表示できる |
| F-IS-07 | 現在位置のソースコードを表示できる（`list`） |
| F-IS-08 | 指定アドレス付近を逆アセンブルして表示できる（`dis`） |
| F-IS-09 | ロードされたシンボル一覧を表示できる（`syms`） |
| F-IS-10 | 行番号とアドレスのマッピングを表示できる（`lines` / `lists`） |

### 4.4 変数書き換え

| ID | 要件 |
|---|---|
| F-WR-01 | 変数・構造体メンバ・配列要素に値を代入できる（`set`） |

### 4.5 メモリリーク検査

| ID | 要件 |
|---|---|
| F-ML-01 | `MallocStackLogging` 環境変数を有効にして debuggee を起動できる |
| F-ML-02 | macOS の `leaks` ツールを呼び出して未解放メモリを報告できる |

### 4.6 設定

| ID | 要件 |
|---|---|
| F-CF-01 | 構造体・配列の pretty print を有効/無効にできる |
| F-CF-02 | 式評価の言語を設定できる（auto / c / c++ 等） |
| F-CF-03 | MallocStackLogging を次回 run から有効にする設定ができる |

### 4.7 ヘルプ

| ID | 要件 |
|---|---|
| F-HP-01 | コマンド一覧を表示できる |
| F-HP-02 | 特定コマンドの使用方法を表示できる |

---

## 5. 非機能要件

| ID | 要件 |
|---|---|
| NF-01 | 対象アーキテクチャは ARM64（Apple Silicon）および x86_64 |
| NF-02 | C11 準拠のソースコードで実装する |
| NF-03 | コンパイラは clang を使用し、`-Wall -Wextra -Wpedantic` で警告なしでビルドされること |
| NF-04 | debugger 自身がクラッシュした場合でも debuggee を zombie として残さないこと |
| NF-05 | `task_for_pid` 権限のため、コード署名とエンタイトルメント（`com.apple.security.cs.debugger`）が付与されること |
| NF-06 | シンボル情報がない実行ファイルでも最低限のアドレスベース操作ができること |

---

## 6. 外部インタフェース

### 6.1 コマンドライン

```
cdbg <program> [args...]
```

### 6.2 依存外部コマンド

| コマンド | 用途 | 必須 |
|---|---|---|
| `nm` | シンボルテーブル読み取り | 任意（フォールバックあり） |
| `dwarfdump` | ローカル変数情報読み取り | `print` / `set` に必要 |
| `leaks` | メモリリーク検査 | `leaks` コマンドに必要 |
| `codesign` | デバッガへの権限付与 | ビルド時に必要 |

### 6.3 OS API

| API | 用途 |
|---|---|
| `ptrace` | プロセストレース、メモリ読み書き（フォールバック） |
| `mach_vm_read` / `mach_vm_write` | メモリ読み書き（主経路） |
| `task_for_pid` | 対象プロセスの Mach タスクポート取得 |
| `thread_get_state` / `thread_set_state` | レジスタ読み書き |
| `mach_vm_region_recurse` | ASLR スライド算出 |

---

## 7. 制約・前提

- デバッグ対象は macOS ネイティブの 64bit Mach-O 実行ファイルに限る
- マルチスレッドプログラムはプライマリスレッド（スレッドリスト先頭）のみ操作する
- ブレークポイントは最大 64 個
- コマンドバッファは最大 256 バイト
- シンボル名は最大 256 文字、ファイルパスは最大 512 文字
